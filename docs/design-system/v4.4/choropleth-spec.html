3Token bindings · light / dark

Dark values follow the B1 derivation rules. One deliberate inversion: the dark sequential ramp reverses lightness order (seq.1=teal-900 → seq.5=teal-100) so higher values stay the brighter mark on a dark surface. Band map fills are theme-stable (the 500-weight band hues already pass ≥3:1 against both surfaces).

4Small-jurisdiction chip strip

Malta, Luxembourg, Cyprus and Liechtenstein can never be honest hit targets on a Europe map. Rule: after layout, any jurisdiction whose rendered bounding box has a minimum dimension < 24px is additionally rendered as a chip in a strip below the map (visible in the demo above, Europe view). Chips are full interaction peers: hover highlights the map region, focus/click select it, and they carry the same fill swatch. The map path stays rendered and colored — the chip adds reachability, it doesn't replace geography. The strip renders only when at least one region trips the threshold; chips order by data value descending. Demo note: the 110m source this page fetches omits Malta's and Liechtenstein's geometry outright; they still appear as chips and table rows because the contract is data-driven — any jurisdiction in data without renderable geometry is chip-only. The production 50m build (D-116) includes their geometry.

5Accessibility contract

Table fallbackVisually-hidden <table> linked via aria-describedby on the role="img" svg — jurisdiction · value · band per row, sorted by value. Sibling of the figure (never inside it) so table semantics survive. Products MUST NOT strip it (PLATFORM-STANDARDS §9.3). The demo's table is live — inspect #map-table-host.
KeyboardThe map is one tab stop; arrow keys rove between jurisdictions (sorted by value), Enter/Space toggles selection, Esc clears. Chips are native buttons in the normal tab order. Demo simplification: every region is a tab stop; production uses the roving pattern.
LabelsEach region: role="graphics-symbol" + aria-label="{name}, {value} {unit}, {band} band". Tooltip is role="status", supplementary only — never the sole carrier of the value.
Color independenceBand/value is always recoverable from the table + labels + legend; never from fill alone. Forced-colors per §2.
MotionOnly stroke transitions (120ms), gated on prefers-reduced-motion / [data-motion] (R13). No entrance animation, no pan/zoom easing in v4.4 (zoom is out of scope).

6Implementation contract — for Claude Code

Locked decisions D-115…D-119 (CEO, 10 June 2026) and the build pipeline. No gray areas intended; anything genuinely undecided follows the nearest analogue and lands in brief §13.

IDDecisionLocked choice
D-115GranularityTwo packaged geographies: geo="europe" — country-level EU-27 + IS/LI/NO + UK (31 jurisdictions, ISO-3166 alpha-3 ids); geo="world" — six continents (Antarctica excluded), slug ids. Same component, same contract.
D-116Geometry sourceNatural Earth (public domain — no attribution requirement). Europe from 1:50m admin-0, world from 1:110m admin-0 merged on the CONTINENT attribute. France/Norway clipped to the European frame (lon −25…32, lat 34…72) so overseas territories don't distort the fit.
D-117Encoding modesBoth: mode="band" (5-band, band.* contract) and mode="seq" (continuous → quantised to chart.seq.1..5; default quintiles, optional thresholds prop). chart.null for no-data in both.
D-118Small jurisdictionsChip strip per §4; threshold = min bbox dimension < 24px at rendered size, re-evaluated on resize.
D-119Asset pipelinePre-projected at build time — no runtime projection dependency (keeps the package zero-dep like the MCP server). tools/charts/build-geo.mjs: Natural Earth → filter/merge → simplify (mapshaper, visvalingam ~10%) → project (azimuthal equal-area fit for europe, natural-earth for world) → quantised planar TopoJSON in data-viz/geo/{europe,world}.json. Budget: ≤ 80KB gzipped combined; CI fails over budget. Runtime renders plain path data.
