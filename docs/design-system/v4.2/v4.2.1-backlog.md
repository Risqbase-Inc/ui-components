## Sonification (audit U1.7)

**Status:** Deferred to v4.3 unless a v4.2.1 spec patch reinstates it.

The original `plan.md` U1.7 listed "sonification optional" as part of
chart accessibility, but `spec.md` §8.5 specifies the five mandatory
provisions (per-mark `aria-label`, reduced-motion, forced-colors,
data-table fallback, screen-reader narrative) without sonification —
the sub-item was silently dropped (audit row U1.7 flag). Per the CEO
directive to **implement the spec as presented**, sonification stays
out of v4.2 implementation. If a v4.2.1 spec patch reinstates it (with
a defined audio-cue model, volume control, and
`prefers-reduced-motion`-aware fallback) it ships in v4.3. Otherwise it
remains a v4.3+ candidate. Owner on reinstatement: Elena + Priya.

## Motion tokens enumeration (closed in v4.2)

**Status:** Resolved in `s1/cleanup-2-design` (this PR).

`implementation-plan.md` §8 item 3 listed motion-token enumeration as a
v4.2.1 patch backlog item. Closed early in v4.2 to avoid two rounds of
churn — `tokens/primitive/duration.json` and
`tokens/primitive/easing.json` now ship the four
`duration.motion.{instant,reactive,considered,celebratory}` values
(0ms / 100ms / 300ms / 500ms) and three
`easing.{standard,emphasised,decelerated}` curves referenced (but not
enumerated) in spec §15.3 and §15.5. Spec §15.3 example
`duration.motion.reactive` now resolves; `duration.gauge.reveal` is
preserved as an alias to `motion.celebratory` (both = 500ms). The
v4.2.1 spec patch should retroactively absorb these values into §5
(Motion) and §15.5 ("v4.1 retained verbatim") so the spec text matches
the token catalogue. Verify against AA contrast / reduced-motion
guidance in §8.5.3 — `motion.instant` is the canonical reduced-motion
fallback.
