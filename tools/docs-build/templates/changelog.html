Versioning convention

The package and the spec version separately. @risqbase-inc/ui-components is semver:

  • Major — breaking API change in a shipped component (renamed prop, removed variant, type-narrowing). Reviewed for migration cost across all three consumers before merge.
  • Minor — additive: new components, new tokens, new variants.
  • Patch — bug fix, contrast remediation, doc string fix. No new tokens; no new variants.

The spec versions semantically but separately. v4.x is the current ladder. A spec point release doesn't always trigger a package version bump — sometimes the spec lands ahead of code.
