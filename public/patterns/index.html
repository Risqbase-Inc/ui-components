Recipes

Patterns

Composite recipes — multi-component arrangements where the visual layout, the voice / tone, the accessibility behaviour, and the state transitions are defined as a single unit. Authored under spec §8.17 (composite chart recipes) and §20 (interaction patterns).

Live recipes

The first three recipes shipped in v4.2.1 A3:

§8.17.1

Risk-band dashboard tile

Metric card + sparkline + gauge + band-coloured rule. Used on overview dashboards. Voice template: "{count} {entity} now sit in the {band} band — {delta} since {period}."

§8.17.2

Compliance posture composite

Stacked horizontal bar (composition) + outlier dot annotations + IRIS-narrated summary text. Used on compliance audit overview surfaces.

§8.17.3

Vendor-risk scatter + flag

Scatter (correlation between risk-score and time-since-review) with flagged vendors elevated via Citation chip annotations.

Recipe shape (per spec §20.0)

Every recipe ships with five required sections — landing as MDX in docs/design-system/v4.2.1/recipes/<name>.md:

  • Anatomy — every component named, in arrangement order.
  • Tokens consumed — exhaustive list with semantic role (no primitive references).
  • Voice examples — paired templates at compact / default / comfortable density (per voice_examples binding §20.0.1).
  • Accessibility contract — ARIA bindings, keyboard, screen-reader narrative, reduced-motion fallback, forced-colors fallback.
  • States — empty / loading / error / partial / live (≥5 enumerated).
