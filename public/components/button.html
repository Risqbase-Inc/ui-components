Components / core / Button

Button

The primary action affordance. Three variants - primary, secondary, ghost - and three sizes (sm, md, lg). On role tokens; themed through CSS variables; identical pixels to the v1.x release.

stable v1.x AA verified

Overview

Variants

One primary action per region (§21.8). Secondary & ghost are supporting actions.

primary
secondary
ghost

Sizes

Hit target is ≥44×44 at md and lg per §5.8.1. sm reserves for table-cell actions and inline contexts where the surrounding text-size justifies the smaller target.

sm · 28h
md · 36h (default)
lg · 44h

States

Default · hover · focus-visible · active · disabled. :focus-visible shows the indigo ring; mouse focus is suppressed via :focus:not(:focus-visible) (§5.8.1).

default
hover
focus-visible
disabled

Across themes

Anatomy

Anatomy uses §7.0 nouns. The button has no chrome, no rail, no divider - keeping it minimal is the point.

Accessibility

ConcernSpecification
Element<button type="button"> when no href; <a> when present. Never <div role="button">.
Accessible nameDerived from children. Icon-only buttons require aria-label.
Keyboard activationEnter + Space trigger the button. Tab moves focus per §5.8.
Focus ring:focus-visible shows 2px indigo-500 ring at 2px offset. :focus:not(:focus-visible) suppresses mouse focus.
Hit target≥44×44px at md and lg (WCAG 2.5.5).
Disabled statearia-disabled="true" when interactive but not actionable. disabled attribute when not focusable.
Forced colorsBorder resolves to ButtonText; surface to ButtonFace; focus to Highlight. Pre-styled by OS.
Reduced motionTransitions on hover / focus collapse to instant under prefers-reduced-motion: reduce.

Do & don't

✓ Do

One primary action per region. Sentence case. Verb + object. The button names the work it does.

✗ Don't

All-caps shouts. "OK" doesn't name the action. Two primaries side-by-side: the reader has no signal which is the right path.

Code

Import

import { Button, PrimaryButton, SecondaryButton, GhostButton } from '@risqbase-inc/ui-components';
// or the per-domain sub-path:
import { Button } from '@risqbase-inc/ui-components/core';

Basic usage

<Button variant="primary" size="md" onClick={handleSave}>
  Save assessment
</Button>

<Button variant="secondary" href="/cancel">
  Cancel
</Button>

<Button variant="ghost" size="sm" onClick={() => setOpen(false)}>
  Skip
</Button>

Shortcut components

<PrimaryButton>Sign up</PrimaryButton>
<SecondaryButton>Cancel</SecondaryButton>
<GhostButton>Skip</GhostButton>
