Tokens

Brand colour

The brand is indigo-600 (#4F46E5). Used for CTAs, links, primary actions, focus rings, brand marks. Hover state is indigo-700 (#4338CA). No other blue tones in the system.

Themes

The base value set is the light theme — every primitive / semantic / component token has its canonical value defined for light. The other three themes are override-only files: any key absent inherits the light value.

ThemeSelectorOverridesWhere
Light:root—tokens/primitive/*.json + tokens/semantic/*.json + tokens/component/*.json
Dark[data-theme="dark"]47 entriestokens/themes/dark.json · view canvas
HC (forced-colors)@media (forced-colors: active)90 entriestokens/themes/hc.json
Print@media print~70 entriestokens/themes/print.json + tokens/component/print-patterns.json

Dark and print themes carry different hex values for the same semantic keys. HC theme uses CSS system colour keywords (Canvas, CanvasText, Highlight) so the user's OS palette wins.

Token JSON (Style Dictionary input)

// tokens/semantic/color.json
{
  "color": {
    "action": {
      "primary": {
        "$value": "{color.brand.indigo.600}",
        "$type": "color",
        "$description": "Primary action surface — CTAs, primary buttons",
        "$extensions": {
          "com.risqbase.role": "semantic",
          "com.risqbase.figma": {
            "collection": "semantic",
            "mode": "light",
            "variable": "color/action/primary",
            "scopes": ["ALL_FILLS", "SHAPE_FILL", "STROKE_COLOR", "TEXT_FILL"]
          }
        }
      }
    }
  }
}

Token shape contract

Every leaf in the token tree must have:

  • $value — string or number. Aliases use {dotted.path} syntax.
  • $type — one of color, dimension, duration, cubicBezier, asset.
  • $description — non-empty human-readable description.
  • $extensions["com.risqbase.role"] — primitive | semantic | component.
  • $extensions["com.risqbase.figma"] — { collection, mode, variable, scopes[] } (scopes optional on primitive).

Enforced by npm run lint:tokens in CI. See tools/tokens-build/lint.js.
